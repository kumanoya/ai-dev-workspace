# commit — 日本語コミット規約と手順

現在の未コミット変更を1件のコミットにまとめる手順の権威。メインセッション（ユーザーの明示依頼時）と committer サブエージェント（自律の区切りコミット時）の両方がこの手順に従う。

## 実行経路の確認（最初に判断する）

- **ユーザーから明示的にコミットを依頼された**（`/commit`、「コミットして」等）→ このままメインセッションで以下の手順を実行する。
- **実装作業の流れの中で、意味的な区切りとして自律的にコミットしようとしている** → このスキルを自分で実行せず、**committer サブエージェント**（Agent ツール、`subagent_type: committer`）に委譲する。`workSummary`（作業内容の要約1〜3行）を会話の文脈から自分で書いて渡す（エージェントに diff から推測させない）。

## 手順（コスト配慮: ツール呼び出しは最小限に）

1. `git branch --show-current` と `git status --short` を**1回**で確認する（`&&` で連結してよい）。
   - ブランチが `main` なら**何もせずに中断**し、フィーチャーブランチの作成を提案する。
   - 変更が無ければ中断する。
2. 差分の把握は `git diff --stat`（+ 未追跡ファイルは status で把握）を基本とし、メッセージ作成に内容確認が必要なファイルだけ `git diff -- <path>` を個別に見る。**全文 diff の一括取得はしない**。
3. コミット対象のパス指定があればそのパスのみ、なければ関連する変更を `git add` する。`.env` や認証情報らしきファイルが含まれていたらステージせず、報告する。
4. 下記のメッセージ規約に従い、heredoc でコミットする（`git commit -m "$(cat <<'EOF' ... EOF)"`）。

## メッセージ規約

- タイトル・本文とも**日本語**（英語のみ不可）。

```
<種別>: <変更内容の要約>

- 変更点
- 理由・影響範囲

Co-Authored-By: Claude <実行モデル名> <noreply@anthropic.com>
```

- 種別の例: 追加 / 修正 / 更新 / 削除 / リファクタリング / ドキュメント
- Co-Authored-By の実行モデル名は**実際にコミットを作成したモデル**（例: `Claude Haiku 4.5`）を入れる

## 制約

- **push・pull・fetch・merge・rebase・amend・reset・checkout は禁止**。コミット作成のみ（push は人間が差分確認後に行う。`git push` は `.claude/settings.json` の `deny` でブロック済み）。
- `main` ブランチへのコミット禁止（手順1で中断）。
- ステージ済みの他の変更を巻き込まない: 対象パスの指定があるときはそのパス以外を `git add` しない。
