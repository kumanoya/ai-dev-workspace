cspell の検出結果を起点に、スペルミスを自律的に解消する。VS Code の Code Spell Checker 拡張で人間が波線に気づき、修正は Claude Code が担う運用の「修正」側を担当する。

**独立タスクとして（タスクの切れ目・新規セッションで）実行する**こと — コマンドレベルのモデル切替（Haiku）は実行中セッションの履歴を読み直すため、長い実装セッションの途中に挟まない。

## 検査対象の方針

**データ系（mock / dummy / sample）はスペルチェックの対象外。** ダミーの人名・ID・サンプル値には「正しいスペル」が存在しない（固有名詞・ランダム文字列）ため、辞書に積むと肥大化し、本物のタイポを覆い隠す。

- ダミーデータは専用ファイルに隔離し、`cspell.json` の `ignorePaths` で除外する（このプロジェクトでは `**/src/data/**`）。新しいダミーデータも原則そこへ集約する。
- 本体コード（型定義・コンポーネント）に少数のダミー値が紛れている場合は、**グローバル辞書に積まず**、その箇所に `// cspell:ignore <語> -- 理由` を付けて局所的に無視する（例: `src/types/index.ts` の JSDoc 例示・`UserCreatePage.tsx` の担当者初期値）。

## 手順

1. **検出**: スペルチェックを実行して指摘一覧を得る。
   - 引数 `$ARGUMENTS` があればその範囲に絞る: `npx cspell --no-progress "$ARGUMENTS"`
   - 無ければプロジェクト全体: `pnpm run lint:spell`
   - 設定の権威は **ルートの `cspell.json`**（`words` / `ignoreWords` / `ignorePaths` / `dictionaries`）。

2. **トリアージ**: 各指摘を必ず該当箇所を `Read` で確認したうえで、次のどれかに分類する。**コードを見ずに機械的に置換しない。** 辞書は最小限を保ち、事前に語を盛らない（実際に引っかかった時だけ足す）。
   - **本物のタイポ** → ファイルを `Edit` で修正する。
     - 例: `Placefolder`→`Placeholder` / `Hoverd`→`Hovered` / `Desabled`→`Disabled` / `folse`→`false`
     - 変数名・プロパティ名・公開API名の変更は影響範囲を `Grep` で確認し、参照箇所もまとめて直す。
     - 文言・意味を変えない。周囲のコードの命名・スタイルに合わせる。

   - **データ系のダミー値**（人名・ID・サンプル）→ 上の「検査対象の方針」に従う。`src/data/**` 等へ隔離できるなら隔離、本体に紛れた局所値は `// cspell:ignore` で対処。**辞書には登録しない。**

   - **再出現する正当なドメイン用語・固有名詞・略語** → `cspell.json` の `words` に追加する。
     - 例: 業務ドメインの語彙コード・略語、製品・プロジェクト名、フォント名、辞書に無い英単語（`toggleable` 等）。
     - 判断基準は「アプリ内で語彙として繰り返し正当に出現するか」。一回限りの値は `words` に入れない。

   - **意味のないコード/ID・固定ノイズ（Figma レイヤーID・原文タイポ・一回限りの連結コード）** → `cspell.json` の `ignoreWords`、または出現が局所的なら対象ファイルに `// cspell:ignore <語>` を付ける。
     - 例: Figma レイヤーID のランダム文字列、`Desabled` / `Hoverd`（Figma 原文のタイポ）、一回限りの連結コード文字列。
     - specs は Source of Truth。Figma 由来の ID・原文は「正」なので**書き換えない**。無視登録で対処する。

   迷う語はユーザーに分類を確認する。勝手にコード側を書き換えるより無視登録のほうが安全。

3. **再実行**: 修正・登録後に再度スペルチェックを走らせ、**Issues found が 0 になる**まで 1〜2 を繰り返す。

4. **報告**: 結果を表で報告する。
   - 「修正したタイポ」（ファイル:行 / 修正前→修正後）
   - 「`words` に登録した用語」
   - 「無視登録した識別子」

## 注意

- 日本語など非ラテン文字は cspell の対象外。CJK を語として直そうとしない。
- `cspell.json` を編集したら `words` / `ignoreWords` はアルファベット順を保ち、重複を作らない。辞書は最小限に保ち、不要になった語は消す。
- このコマンドはコミットしない（差分はユーザーが確認する）。
