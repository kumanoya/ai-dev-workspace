現在のセッションの作業状態を引き継ぎメモとして `tmp/handoff/` に書き出す。「1機能=1セッション」運用の要: メモを書いたら `/clear` し、次のセッションは肥大した会話履歴ではなくこのメモ＋git から文脈を復元する。

## 手順

1. 書き出し先を決める: `tmp/handoff/<トピック>-<YYYYMMDD>.md`。トピックは `$ARGUMENTS`、なければ現在のブランチ名から。ディレクトリが無ければ作る（`tmp/` は gitignore 済み）。
2. 以下の構成でメモを書く。**次のセッションの Claude が読者**なので、会話内でしか通じない略語・言い回しは使わない:
   - **目的**: このタスクは何のためか（1〜2行）
   - **完了したこと**: 変更内容の要点。コミット済みなら該当コミットハッシュを併記
   - **未完了・次の一手**: 具体的な作業単位で。着手済みで中途半端なファイルがあれば明記
   - **決定事項・制約**: セッション中に決めたこと・ハマった罠・やらないと決めたこと（次セッションが再調査・再議論しないため）
   - **関連ファイル**: 主要な対象ファイルのパスと1行説明。specs に書いてあることはパス参照で済ませ、内容を転記しない
3. 未コミットの変更が残っていれば、先に `/commit` を促す（メモは「git に無い情報」だけを担う）。
4. メモのパスをユーザーに伝え、`/clear` → 新セッションで「`tmp/handoff/xxx.md` を読んで続きから」と指示すればよいことを案内する。

## 制約

- メモは1タスク1ファイル。既存メモの続きなら同じファイルを更新する。
- 会話ログの転記・生 diff の貼り付けはしない（git で復元できる情報は書かない）。
